Manually Generating Certificates

The page on Kubernetes' official documentation provides detailed instructions on how to manually generate certificates for client certificate authentication in a Kubernetes cluster. Three methods are discussed: using `easyrsa`, `openssl`, and `cfssl`.

easyrsa

  1. Download and initialize easyrsa3.
  2. Generate a new Certificate Authority (CA).
  3. Generate the server certificate and key.


openssl

  1. Generate a `ca.key` with 2048 bits.
  2. Create a `ca.crt` based on `ca.key`.
  3. Generate a `server.key` with 2048 bits.
  4. Create a config file for generating a Certificate Signing Request (CSR).
  5. Generate the certificate signing request based on the config file.
  6. Generate the server certificate using `ca.key`, `ca.crt`, and `server.csr`.


cfssl

  1. Download and prepare the command-line tools.
  2. Create a directory to hold the artifacts and initialize cfssl.
  3. Generate CA key (`ca-key.pem`) and certificate (`ca.pem`).
  4. Generate the key and certificate for the API server.
