Downward API




Two Methods of Exposure

Pod and container fields can be exposed to a running container in two ways: as environment variables and as files in a special volume type called the "downward API."


Low Coupling

The downward API allows containers to have information about themselves or the cluster without being tightly coupled to Kubernetes. This is useful for legacy applications that rely on specific environment variables.


Available Fields

Only certain Kubernetes API fields are available through the downward API. These include metadata like the pod's name, namespace, and unique ID, as well as resource requests and limits like CPU and memory.


Fallback for Resource Limits

If CPU and memory limits are not specified, the `kubelet` defaults to exposing the maximum allocatable value for these resources based on node allocatable calculations.
